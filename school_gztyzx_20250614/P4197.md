**【题目背景】**

蒜算机是一种量子计算机！

**【题目描述】**

在蒜算机的操作系统中，有两种指令，分别是 `+ x` 和 `-`，其中 $x$ 是一个整数。

对于一个指令序列 $S$，蒜算机会这样计算它的结果 $calc(S)$：

- 依次遍历 $S$ 中的指令，并维护一个可重集 $T$；
- 对于一条指令，若其为 `+ x`，就将 $x$ 加入 $T$；否则删除 $T$ 中最小的整数，若 $T$ 为空则不执行这条指令；
- $calc(S)$ 即为执行完操作序列后的 $T$ 中的所有整数的和。

小 C 现在有一个指令序列 $A$，他想求出这个指令序列的所有子指令序列放在蒜算机上运行之后的结果的和。即求 $\sum\limits_{B\text{ 是 }A\text{ 的子指令序列 }}calc(B)$。

小 C 发现这个东西似乎有点大，所以他想求出

$\left(\sum\limits_{B\text{ 是 }A\text{ 的子指令序列 }}calc(B)\right)\mod 998244353$

的值，你能帮帮他吗？

**【输入格式】**

从文件 `opt.in` 中读入数据。

第一行一个整数 $n$，表示 $A$ 的长度。

接下来 $n$ 行每行一条指令，格式为 `+ x` 或者 `-`。

**【输出格式】**

输出到文件 `opt.out` 中。

一行一个整数表示答案。

**【样例输入 1】**

```
4
-
+ 1
+ 2
-
```

**【样例输出 1】**

```
16
```

**【样例输入 2】**

```
15
+ 2432543
-
+ 4567886
+ 65638788
-
+ 578943
-
-
+ 62356680
-
+ 711111
-
+ 998244352
-
-
```

**【样例输出 2】**

```
750759115
```

对于 $20\%$ 的数据，$n\le 100$。

对于 $100\%$ 的数据，$1\le n\le 500$，$1\le x<998244353$。
