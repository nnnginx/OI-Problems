## 问题描述

程序龙精挑细选，找到了一套适合于本次测试难度的试卷，共有 $N$ 个题，其中第 $i$ 个题的难度是 $1\times 2\times 3\times \cdots \times i$ (也可以写作 $i!$)。我们称一套试卷的难度为 $N$ 个题的难度总和。

那么给定 $N$，求这套试卷的难度总和。

然而，总有些人不想写高精度， 所以你只需要算出难度总和对 $M$ 取模后的结果就好了。

## 输入格式

输入只有一行，两个整数 $N$ 和 $M$，中间用一个空格隔开;

## 输出格式

输出共一行一个整数，难度总和对 $M$ 取模的结果;

```input1
4 10007
```

```output1
33
```

## 样例解释

$1 + 1\times 2 + 1\times 2\times 3 + 1\times 2\times 3\times 4 = 1+2+6+24 = 33\% (mod) 10007 = 33$;

## 数据规模

对于 $20\%$ 的数据: $1≤N, M≤10$;

对于 $50\%$ 的数据: $1≤N, M≤1, 000, 000$;

对于 $100\%$ 的数据: $1≤N≤1, 000, 000, 000, 000, 000, 000; 1≤M≤1, 000, 000$;
