## 问题描述

陈帅龙的儿子将要结婚，为了办上一场最盛大的婚礼，陈帅龙几乎倾家荡产，摆了 $n$ 桌宴席，每桌宴席都是由各种美味组成，当然，上菜的顺序肯定是由美味值小的到大的上。

终于，这激动人心的一天终于来了，一道一道的美味顺次摆上了宴席，陈帅龙看着这些美味，忍不住将每桌的一部分偷偷吃掉了（吃掉的部分各不相同），导致每桌的美味总值各不相同。正当他吃完回过神来，才发现闯下了大祸：如果客人看到他们这一桌的美味值不如其它桌，客人就会生气。想到这，陈帅龙忍不住打了一个哆嗦。客人快要来了，重新上新的已经来不及了，陈帅龙灵机一动，想出了一个一个巧妙的改造方案：他决定从其中几个宴席中再吃掉一些美味，使得最终每桌美味总值都一样大，为了不让客人贬低自己，他觉得应该使最后每桌的美味总值都尽可能的高。

任务:请你编写一个程序，根据被吃剩的所有宴席的信息，决定应该吃掉哪些美味才能获得最佳的效果。

## 输入格式

第一行是一个正整数 $n$，表示一共有 $n$ 桌宴席。

以下 $n$ 行每行是一系列非负整数，用一个空格分隔，从左到右、从大到小给出一桌宴席中所有美味的美味值。用 $-1$ 结束。

## 输出格式

一个整数，表示最后宴席的最大可能的美味总值。

如果找不到合适的方案，则输出0。

```input1
2
2 1 -1
3 2 1 -1
```

```output1
3
```

## 数据范围

对于 $100\%$ 的数据，$1 ≤ n ≤ 1000$，一桌宴席中的美味不超过 $100$ 道，每道美味的美味值不超过 $100$
