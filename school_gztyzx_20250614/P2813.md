## 问题描述

南海自古以来就是中国的领土，但是近几年美国一直在南海搅局，试图使用这种拙劣的手段阻碍中国的发展。为了守卫南海，我国海军经过三十多年的发展，现在已经有两艘航空母舰服役，还有多艘万吨级的导弹护卫舰服役守卫我们的家园。最近，我国研发了一种新型的海上雷达拦截系统，用于解决南海存在的各种问题。当雷达打开时，有一定的搜索半径，凡是在搜索半径内的任何攻击都可以成功拦截，即便是隐藏在搜索半径等于 $0$ 处的攻击也可以成功拦截。但是雷达拦截系统存在一个Bug，每艘船的搜索半径每天只能设定一次。而当天使用雷达拦截系统的使用代价就是所有船拦截系统搜索半径的平方和。

某天，雷达拦截系统捕捉到美国和越南的攻击。由于该系统尚不完善，所有只有两套拦截系统投入了工作。如果现在要求拦截所有的攻击，请您帮忙设计个算法计算这一天的最小使用代价。

## 输入格式

第一行包含 $4$ 个整数 $x_1、y_1、x_2、y_2$，表示这两套雷达拦截系统的坐标为($x_1,y_1$)、($x_2,y_2$)。

第二行包含 $1$ 个整数 $N$，表示有 $N$ 个攻击。接下来 $N$ 行，每行两个整数 $x、y$，表示一个攻击的坐标($x,y$)。不同攻击的坐标可能相同。

## 输出格式

输出只有一行，包含一个整数，即当天的最小使用代价。

```input1
0 0 10 0
2
-3 3
10 0
```
```output1
18
```

## 样例1解释

要拦截所有攻击，在满足最小使用代价的前提下，两套系统工作半径的平方分别为 $18$ 和 $0$。 

```input2
0 0 6 0
5
-4 -2
-2 3
4 0
6 -2
9 1
```
```output2
30
```

## 数据范围
对于 $60\%$ 的数据，$1≤N≤100$

对于 $70\%$ 的数据，$1≤N≤1000$

对于 $100\%$ 的数据，$1≤N≤100000$，且所有坐标分量的绝对值都不超过 $1000$。
