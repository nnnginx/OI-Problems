## 问题描述

今天是圣诞节，程序龙送给你 $n$ 科作业，完成第 $i$ 科作业需要花 $a_i$ 单位时间。显然，你可能一天完成不了这么多的作业量，但是你可以今天先用 $m$ 单位时间完成作业，剩下的明天再完成。

已知你完成第 $i$ 科作业，则**每单位时间**将得到 $x_i$ 点开心值，将第 $i$ 科作业留到第二天完成则**每单位时间**可以获得 $y_i$ 点开心值。当然，可以今天完成第 $i$ 科作业的一部分，剩下部分留待明天，但完成第 $i$ 科作业的总时间依然为 $a_i$。

如何使得你今天完成 $m$ 个单位时间的作业，第二天完成 $\sum_{i=1}^na_i-m$ 个单位时间作业后，总开心值最大呢？

## 输入格式

第一行两个整数 $n$ 和 $m$；

接下来 $n$ 行每行三个正整数 $a_i,x_i,y_i$ 意义如上所述。

## 输出格式

输出共一行一个整数，最大开心值。

```input1
2 3
2 1 1
2 1 2
```

```output1
5
```

## 样例解释

$m=3$，即你今天要完成 $3$ 单位时间的作业量。你可以先完成第 $1$ 科作业，需要 $2$ 单位时间，获得开心值为 $2\times 1=2$，再花 $1$ 单位时间完成第 $2$ 科作业，获得开心值为 $1\times 1=1$，所以总时间为 $3$，总开心值为 $2+1=3$

第二天，你对于第 $2$ 科作业还有 $1$ 单位时间的作业未完成，完成第 $2$ 科作业的剩余部分，获得开心值为 $1\times 2=2$，总开心值为 $2$

所以，两天的总开心值为 $5$。

不存在比 $5$ 更大的方案

## 数据规模

对于 $30\%$ 的数据：$1≤n≤10；1≤a_i≤10；$

对于 $60\%$ 的数据：$1≤n≤1,000；1≤a_i≤100；$

对于 $100\%$ 的数据：$1≤n≤100,000；1≤a_i,x_i,y_i≤100,000；m≤\sum_{i=1}^na_i；$
