## 题目描述

CodingCow 将在成都旅游 $q$ 天，已知成都有 $n$ 个景点，每一个景点都有一个类别 $c_i$，有 $m$ 条双向道路链接这 $n$ 和景点使得景点之间互相可以到达。

每一条道路都有一个困难程度 $k_i$，CodingCow 每天会从起点 $x$ 出发做若干次出行。对于第 $i$ 天，CodingCow 对道路的接受程度初始为 $l_i$，每次出行后会增加 $1$，直到接受程度大于 $r_i$ 时，CodingCow 不再出行。注意 CodingCow 能通过通行难度为 $k_i$ 的道路当且仅当此时 CodingCow 的接受程度大于等于 $k_i$。CodingCow 可能一次出行会去很多不同的景点。

对于每一次出行，CodingCow 会获得的愉悦值时出行所到达的景点种类数。相同种类的景点不会增加愉悦值，一天的愉悦值为当天所有出行的愉悦值之和（每一天的时间可以认为是无限的）。

求 CodingCow $1 \sim q$ 天每天的愉悦值。

## 输入格式

第一行四个整数 $n,\,m,\,q,\,x$，分别表示景点数目、道路数目、游玩天数以及每次游玩的起点。

第二行 $n$ 个整数，表示每一个景点的类型 $c_i$。

此后 $m$ 行每行三个整数 $u,\,v,\,w$，表示一条连接 $(u,\,v)$，难度为 $w$ 的道路。

再此后 $q$ 行每行两个整数 $l_i,\,r_i$，表示第 $i$ 天的接受程度区间。

## 输出格式

输出共 $q$ 行，表示每天出行的愉悦值。

## 样例

```input1
3 2 2 1
1 2 3
1 2 2
1 3 3
1 2
1 3
```

```output1
3
6
```

```input2
见附件/tour2.in
```

```output2
见附件/tour2.ans
```

### 数据范围

对于 $40\,\%$ 的数据，$1 \leq n,\,m,\,q,\,k_i,\,c_i,\,l_i,\,r_i \leq 100$。

对于 $60\,\%$ 的数据，$1 \leq l_i,\,r_i,\,k_i \leq 10^5$。

对于 $100\,\%$ 的数据， $1 \leq n,\,m \leq 5\times 10^5$，$1 \leq q \leq 10^5$，$1 \leq c_i \leq 10^3$，$0 \leq k_i,\,l_i,\,r_i \leq 10^9$。

此外，有 $15\,\%$ 的数据满足 $m = n - 1$，均匀分布在各档数据内。
