## 问题描述

石头剪刀布的游戏可谓是家喻户晓。今天小龙闲来无事，写了一个程序与自己进行游戏。

游戏规则：每局双方都可以出三个手势中的一个：石头（rock）、剪刀（scissors）、布（paper）。石头可以赢剪刀，剪刀可以赢布，布可以赢石头，相同则平局。

毕竟小龙是一位资深的程序员，他对随机数研究深入，所以他能知道每一局程序会出什么手势。

玩了一会，小龙觉得这样玩太无聊了，于是增加了一个规则：小龙一开始有一个点数值 $p$ 。每一轮，如果小龙出石头，将不消耗点数；如果出剪刀，则消耗两点点数；如果出布，则消耗五点。

小龙想在有限的点数下赢尽可能多的局数，现在请你求一个方案，使得小龙赢的局数最多。

由于答案可能有很多种，你只需要输出任意一种即可。

## 输入格式

输入第一行两个数 $n$ 和 $p$ ，分别表示游戏进行的局数和小龙的初始点数。

接下来一行， $n$ 个字符串，如果为 `rock`，则表示第 $i$ 局程序会出石头；如果为`scissors`，则表示第 $i$ 局程序会出剪刀；如果为`paper`，则表示第 $i$ 局程序会出布。

## 输出格式

输出一行， $n$ 个数，第 $i$ 个数表示第 $i$ 局小龙会出的手势，如果为`rock`，则表示第 $i$ 局小龙会出石头；如果为`scissors`，则表示第 $i$ 局小龙会出剪刀；如果为`paper`，则表示第 $i$ 局小龙会出布。

```input1
2 2
```

```output1
scissors rock
```

## 样例1解释

找不到任何一种方案小龙可以赢得程序多余两局。此外，`rock scissors` 也是一个可行解。

```input2
5 3
```

```output2
rock rock scissors rock rock
```

## 数据范围：

对于 $30\%$ 的数据，程序每局出的手势相同 。

对于 $100\%$ 的数据， $1 \le n,k \le 10^5$ ,保证输入文件中仅出现`rock`，`scissors`，`paper` 三种单词。
