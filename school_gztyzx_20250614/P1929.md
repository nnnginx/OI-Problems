## 问题描述

程序龙来到了著名的oiClass风景区，这里风光旖旎，名胜无数。为了节约游览时间，程序龙希望确定一条最佳游览路线，希望请你帮助一下他。

假设所有的风景点都集中在一个 $C$ 列 $R$ 行的矩阵中，矩阵的每一元素代表风景点或者障碍物。现在你要寻找一条满足下列条件的最佳旅游路线：

- 这条路线上的每一点结点必须是风景点（即不能为障碍物）
- 每个风景点最多游览一次
- 这条路线必须是连续的相邻风景点的序列（若风景点 $A$ 和 $B$ 分别位于矩阵的位置 $(a_1，a_2)$ 及 $(b_1，b_2)$，且 $|a_1-b_1|+|a_2-b_2|=1$，则风景点 $A$ 和 $B$ 是相邻的）
- 在满足上述条件下，游览的风景点尽可能多

假设任意的两个风景点都有且仅有一条路径（无回路）相连。显然，任意一个风景点都可以作为游览路线的起点或者终点。

## 输入格式

第一行是两个整数 $C$ 和 $R(3≤C，R≤1000)$，表示矩阵的列数和行数。

接下来有 $R$ 行，每行有 $C$ 个字符，每个字符都只能是 `#` 或 `．`，`#` 表示障碍物，`．` 表示风景点。行手行末无多余空格。

## 输出格式

只有一行，输出最佳路线的长度。

```input1
3 3
###
#.#
###
```

```output1
0
```

```input2
6 4
##.##.
#..#..
...#..
...#..
```

```output2
8
```
