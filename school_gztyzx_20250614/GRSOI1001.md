## 题目背景

xiaosi4081 正在去饭堂，脑子里正在思考.....

## 题目描述

饭堂中，每一个菜都有一个**饱腹值** $a_i$ 和**好吃程度** $b_i$。

xiaosi4081 正在思考今天吃什么，但是他有选择困难症，所以他决定把所有菜品全都吃一遍！

但是，他要合理安排一个**品尝美食的顺序**使得他吃完某道菜后的**饱腹程度**最大值最小。

饱腹程度取决于他已经吃过的菜（不计入当前吃的菜）的**饱腹值**和 $weight$ 和当前吃的菜的**好吃程度** $yum$，其计算公式为 $weight-yum$。

形式化的说，吃完第 $i$ 个菜后，xiaosi4081 的饱腹程度为 $(\sum_{j=1}^{i-1}a_j) - b_i$。

显然，面对数量众多的菜品，靠手工计算肯定是不可行的，所以 xiaosi4081 决定让你来帮他计算出这样的最小饱腹程度和其对应的方案，如果你拒绝，他会呼叫上帝让你三天不能吃饭。

**本题开启 Special Judge，你可以输出任意一种满足题意的方案。**

注意，饱腹值可以为负数。

## 输入格式

第 $1$ 行有一个数 $n$ 表示菜品数量。

接下来 $n$ 行，第 $1+i$ 行有两个数 $a_i$ 和 $b_i$，意义如题中所示。

## 输出格式

输出共两行，第一行表示最小的最大饱腹程度，第二行表示吃饭的方案（输出菜品原顺序的一组排列）。

## 样例 #1

### 样例输入 #1

```
3
1 1
2 2
3 3
```

### 样例输出 #1

```
0
1 2 3
```

## 提示

$1\leq n \leq 10^5$

$1\leq a_i,b_i \leq 2\times 10^9$
