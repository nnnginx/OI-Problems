CKV写字是出了名的丑，仅好于某教练。于是某教练让CKV去练字，在方方正正的格子里面写数字。比如，数字$2$，写在$5\times 4$的格子里面，就是：
![image](./1780/file/TWCM1uKLnu98YyjJQmvir.png)

然而CKV极其淘气，在这个$5\times 4$的格子里面，把数字$0$~$9$写了好多好多遍，这个格子都快被描黑了。现在，某教练拿到了CKV描黑了的$5\times 4$的纸，并且通过某技术检测到了每个格子被写的次数。问：CKV到底在这个格子里面写了多少个$1,2,...,9,0$？

**如果有多解，输出$1$的个数最小的那一个，如果依然多解，输出$2$的个数最小的那一个，以此类推，最后输出$0$的个数最小的那一组解**。

### 输入格式

一共输入$5$行，每行$4$个整数，代表CKV该格子的涂写次数。

### 输出格式

输入一行共$10$个整数，分别表示$1$~$0$被涂写的次数，**不需要用空格隔开**。

### 样例输入1

```
3 3 3 4
2 0 0 2
3 3 3 4
1 0 0 4
3 3 3 4
```

### 样例输出1

```
1010110000
```

### 样例解释

你一定会喜欢下面这一段东西的

```c++
...1222233334..4555566667777888899990000

...1...2...34..45...6......78..89..90..0

...122223333444455556666...7888899990..0

...12......3...4...56..6...78..8...90..0

...122223333...455556666...7888899990000
```

### 数据分布

对于30%的测试数据，满足每个数字最多被CKV写了$3$遍。

对于60%的测试数据，满足每个数字最多被CKV写了$6$遍。

对于100%的测试数据，满足每个数字最多被CKV写了$9$遍。
