# Description
uim神哪玫搅uoi的ra（镭牌）后，立刻拉着基友小A到了一家......餐馆，很低端的那种。

uim指着墙上的价目表（太低级了没有菜单），说：“随便点”。

不过uim由于买了一些辅（e）辅（ro）书，口袋里只剩 M 元（M ≤ 10000）。餐馆虽低端，但是菜品种类不少，有 N 种（N ≤ 100），第 i 种卖 ai 元（ai ≤ 1000）。由于是很低端的餐馆，所以每种菜只有一份。

小A奉行“不把钱吃光不罢休”，所以他点单一定刚好把uim身上所有钱花完。他想知道有多少种点菜方法。

由于小A肚子太饿，所以最多只能等待 1 秒。

# Format

## Input
第一行是两个数字，表示 N 和 M。

第二行起，N 个正整数 ai（可以有相同的数字，每个数字均在 1000 以内）。

## Output
一个正整数，表示点菜方案数，保证答案的范围在 int 之内。

# Samples

```input1
4 4
1 1 2 2
```

```output1
3
```
