## 题目描述

`小 C` 正在玩一个移球游戏，他面前有 $\red{n + 1}$ 根柱子，柱子从 $\red{1 \sim n + 1}$ 编号，其中 $\red 1$ 号柱子、$\red 2$ 号柱子、$\dots$、$\red n$ 号柱子上各有 $\red m$ 个球，它们自底向上放置在柱子上，$\red{n + 1}$ 号柱子上初始时没有球。这 $\red{ n \times m  }$ 个球共有 $\red n$ 种颜色，每种颜色的球各 $\red m$ 个。

初始时一根柱子上的球可能是五颜六色的，而`小 C` 的任务是将所有同种颜色的球移到同一根柱子上，这是唯一的目标，而每种颜色的球最后放置在哪根柱子则没有限制。

`小 C` 可以通过若干次操作完成这个目标，一次操作能将一个球从一根柱子移到另一根柱子上。更具体地，将 $\red x$ 号柱子上的球移动到 $\red y$ 号柱子上的要求为：

- `1`. $\red x$ 号柱子上至少有一个球；
- `2`. $\red y$ 号柱子上至多有 $\red{m - 1}$ 个球；
- `3`. 只能将 $\red x$ 号柱子最上方的球移到 $\red y$ 号柱子的最上方。

`小 C` 的目标并不难完成，因此他决定给自己加加难度：在完成目标的基础上，使用的操作次数不能超过 $\red{820000}$。换句话说，`小 C` 需要使用至多 $\red{820000}$ 次操作完成目标。

`小 C` 被难住了，但他相信难不倒你，请你给出一个操作方案完成`小 C` 的目标。合法的方案可能有多种，你只需要给出任意一种，题目保证一定存在一个合法方案。

## 输入格式

第一行两个用空格分隔的整数 $\red{n，m}$。分别表示球的颜色数、每种颜色球的个数。

接下来 $\red n$ 行每行 $\red m$ 个用单个空格分隔的整数，第 $\red i$ 行的整数按自底向上的顺序依次给出了 $\red i$ 号柱子上的球的颜色。

## 输出格式

本题采用自定义校验器（`Special Judge`）评测。

你的输出的第一行应该仅包含单个整数 $\red k$，表示你的方案的操作次数。你应保证
$\red {0\le k\le 820000}$。

接下来 $\red k$ 行每行你应输出两个用单个空格分隔的正整数 $\red {x, y}$，表示这次操作将 $\red x$ 号柱子最上方的球移动到 $\red {y}$ 号柱子最上方。你应保证 $\red {1\le x, y \le n + 1}$ 且 $\red {\red{x \neq y}}$。

## 样例

#### 输入

```C
2 3
1 1 2
2 1 2
```

#### 输出样例

```C
6
1 3
2 3
2 3
3 1
3 2
3 2
```

## 提示

柱子中的内容为：按自底向上的顺序依次给出柱子上每个球的颜色。

|  操作  | $\red 1$ 号柱子 | $\red 2$ 号柱子 | $\red 3$ 号柱子 |
| :----: | :--------: | :--------: | :--------: |
|  初始  | $\red{1\ 1\ 2}$  | $\red{2\ 1\ 2}$  |            |
| $\red {1\ 3}$ |   $\red{1\ 1   }$   | $\red{2\ 1\ 2 }$  |    $\red 2$     |
| $\red {2\ 3}$ |   $\red{1\ 1   }$   | $\red{2\ 1    }$   |   $\red {2\ 2}$   |
| $\red {2\ 3}$  |  $\red{1\ 1   }$   | $\red{2       }$     | $\red {2\ 2\ 1}$  |
| $\red {3\ 1}$ |   $\red{1\ 1\ 1}$  |  $\red{2       } $     |   $\red {2\ 2}$   |
| $\red {3\ 2}$ |   $\red{1\ 1\ 1} $  | $\red{2\ 2    }$   |    $\red 2$     |
| $\red {3\ 2}$  |  $\red{1\ 1\ 1}$  |  $\red{2\ 2\ 2 }$  |            |

| 测试点编号  | $\red{n\le}$ | $\red{m\le}$ |
| :---------: | :----: | :----: |
| $\red{1\sim 2 }$  |  $\red {2 }$   |$\red{20 }$  |
| $\red{3\sim 5 }$  |  $\red {10}$  | $\red{20 }$  |
| $\red{6\sim 8 }$   | $\red {50}$  | $\red{85 }$  |
| $\red{9\sim 14}$  |  $\red {50}$  | $\red{300}$  |
| $\red{15\sim 20}$ |  $\red {50}$  | $\red{400}$  |

对于所有测试点，保证 $\red{2\le n\le 50}$，$\red {2\le m\le 400}$。

#### 校验器

为了方便选手测试，在下发文件中我们下发了 `checker.cpp` 文件，选手可以编译该程序，并使用它校验自己的输出文件。但请注意它与最终评测时所使用的校验器并不完全一致。你也不需要关心其代码的具体内容。

编译命令为：`g++ checker.cpp -o checker -std=c++11`。

`checker` 的使用方式为：`checker <inputfile> <outputfile>`，参数依次表示输入文件与你的输出文件。

若你输出的数字大小范围不合法，则校验器会给出相应提示。若你的输出数字大小范围正确，但方案错误，则校验器会给出简要的错误信息：

1. `A x`，表示进行到第 $\red x$ 个操作时不合法。
2. `B x`，表示操作执行完毕后第 $\red x$ 个柱子上的球不合法。
   若你的方案正确，校验器会给出 `OK`。
