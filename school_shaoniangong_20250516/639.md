## 题目描述

若一个数（首位不为零）从左向右读与从右向左读都一样，我们就将其称之为回文数。

例如：给定一个十进制数 $\red{56}$，将 $\red{56}$ 加 $\red{65}$（即把 $\red{56}$ 从右向左读），得到 $\red{121}$ 是一个回文数。

又如：对于十进制数 $\red{87}$：

- `STEP1`：$\red{87+78=165}$
- `STEP2`：$\red{165+561=726}$
- `STEP3`：$\red{726+627=1353}$
- `STEP4`：$\red{1353+3531=4884}$

在这里的一步是指进行了一次 $\red N$ 进制的加法，上例最少用了 $\red 4$ 步得到回文数 $\red{4884}$。

写一个程序，给定一个 $\red N$（$\red{2 \le N \le 10}$ 或 $\red{N=16}$ 进制数 $\red M$（$\red{100}$ 位之内），求最少经过几步可以得到回文数。如果在 $\red{30}$ 步以内（包含 $\red{30}$ 步）不可能得到回文数，则输出 `Impossible!`。

## 输入格式

两行，分别是 $\red{N,M}$。

## 输出格式

如果能在 $\red{30}$ 步以内得到回文数，输出格式形如 `STEP=ans`，其中 $\red{ans}$ 为最少得到回文数的步数。

否则输出 `Impossible!`。

## 样例

#### 输入样例

```c
9
87
```

#### 输出样例

```c
STEP=6
```
