## 题目描述

现在我们的手头有$\red{N}$个软件，对于一个软件$\red{i，}$它要占用$\red{Wi}$的磁盘空间，它的价值为$\red{V_i}$。我们希望从中选择一些软件安装到一台磁盘容量为$\red{M}$计算机上，使得这些软件的价 值尽可能大（即$\red{V_i}$的和最大）。
但是现在有个问题：软件之间存在依赖关系，即软件$\red{i}$只有在安装了软件$\red{j（}$包括软件$\red{j}$的直接或间接依赖）的情况下才能正确工作（软件$\red{i}$依赖软件$\red{j)}$。幸运的是，一个软 件最多依赖另外一个软件。如果一个软件不能正常工作，那么它能够发挥的作用为$\red{0}$。

我们现在知道了软件之间的依赖关系：软件$\red{i}$依赖软件$\red{D_i}$。现在请你设计出一种方案，安装价值尽量大的软件。一个软件只能被安装一次，如果一个软件没有依赖则$\red{D_i=0，}$这时只要这个软件安装了，它就能正常工作。

## 输入格式

第$\red{1}$行：$\red{N, M  （}$$\red{0<=N<=100, 0<=M<=500）}$

第$\red{2}$行：$\red{W_1, W_2, ... W_i, ..., W_n （}$$\red{0<=W_i<=M ）}$


第$\red{3}$行：$\red{V_1, V_2, ..., V_i, ..., V_n  （}$$\red{0<=V_i<=1000 ）}$

第$\red{4}$行：$\red{D_1, D_2, ..., D_i, ..., D_n （}$$\red{0<=D_i<=N, D_i}$≠$\red{i ）}$



## 输出格式

一个整数，代表最大价值。

## 样例

#### 输入样例

```c
3 10
5 5 6
2 3 4
0 1 1
```

#### 输出样例

```c
5
```
