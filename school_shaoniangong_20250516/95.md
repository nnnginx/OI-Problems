## 题目描述

所谓虫食算，就是原先的算式中有一部分被虫子啃掉了，需要我们根据剩下的数字来判定被啃掉的字母。

来看一个简单的例子：

<pre>
 43#9865#045
+  8468#6633
--------------
 44445506978
</pre>

其中`#`号代表被虫子啃掉的数字。

根据算式，我们很容易判断：第一行的两个数字分别是$\red 5$和$\red 3$，第二行的数字是$\red 5$。

现在，我们对问题做两个限制：

首先，我们只考虑加法的虫食算。这里的加法是$\red{N}$进制加法，算式中三个数都有$\red N$位，允许有前导的$\red 0$。

其次，虫子把所有的数都啃光了，我们只知道哪些数字是相同的，我们将相同的数字用相同的字母表示，不同的数字用不同的字母表示。

如果这个算式是$\red N$进制的，我们就取英文字母表的前$\red N$个大写字母来表示这个算式中的$\red 0$到$\red {N-1}$这$\red N$个不同的数字：但是这$\red N$个字母并不一定顺序地代表$\red 0$到$\red {N-1}$。

输入数据保证$\red N$个字母分别至少出现一次。

<pre>
   BADC
+  CBDA
----------
   DCCC
</pre>

上面的算式是一个$\red 4$进制的算式。

很显然，我们只要让$\red{ABCD}$分别代表$\red{0123}$，便可以让这个式子成立了。

你的任务是，对于给定的$\red N$进制加法算式，求出$\red N$个不同的字母分别代表的数字，使得该加法算式成立。

输入数据保证有且仅有一组解。

## 输入格式

输入包含4行。

第一行有一个正整数$\red{N(N<=26)}$，后面$\red 3$行每行有一个由大写字母组成的字符串，分别代表两个加数以及和。

这$\red{3}$个字符串左右两端都没有空格，并且恰好有$\red N$位。

## 输出格式

输出包含一行。

在这一行中，应当包含唯一的那组解。

解是这样表示的：输出$\red N$个数字，分别表示$\red {A，B，C……}$所代表的数字，相邻的两个数字用一个空格隔开，不能有多余的空格。

## 样例

#### 输入数据

```C
5
ABCED
BDACE
EBBAA
```

#### 输出数据

```C
1 0 3 4 2
```
