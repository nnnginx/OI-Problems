## 题目描述

一个闭合的栅栏是平面上的一些不相交的首尾相连的线段形成的多边形，有$\red N$个角(顶点) $\red{(3 < N < 200)}$。 顶点不重合，它以逆时针方式以数组$\red{\{x_i,y_i\}}$,给出$\red{(i=1,2,...,N)}$。

每一对相邻的顶点都是一条栅栏。因此共有$\red N$条栅栏 (定义$\red{xN+1=x1, yN+1=y1}$)。

这里有一个栅栏的例子和一个点$\red x$,$\red y$:

```c
                        * x3,y3
                x5,y5  / \
   x,y *          *   /   \
                 / \ /     \
                /   *       \
          x6,y6*   x4,y4     \
               |              \
               |               \
          x1,y1*----------------* x2,y2
```

请编写一个程序实现下面的任务:

检查输入的顶点列表$\red{\{x_i,y_i\}}$, $\red{i=1,2,...,N}$, 判断它是否为一个合法的闭合栅栏。
找出所有可以被站在点$\red{(x,y)}$处的人所能看到的栅栏(忽略人的高度)，因为有的栅栏会被另外的栅栏所挡住。

只有当存在从$\red{(x,y)}$发射的一条射线第一个穿过栅栏i时，栅栏i是可以被看见的。如果栅栏是平行于目光的，它并不认为是可以看见的。在上面的例子里，线段$\red{[x_3,y_3]-[x_4,y_4], [x_5,y_5]-[x_6,y_6], [x_6-y_6]-[x_1,y_1]}$是可以被$\red{(x,y)}$看见的。

## 输入格式

第$\red 1$行: $\red N$, 表示闭合栅栏的顶点数。

第$\red 2$行: 两个整数$\red x$和$\red y$，表示观测者的位置。两个整数都是$\red{16}$位的。即$\red{2^{16}}$，在$\red{longlong}$或$\red{longint}$范围内。

第$\red{3 \sim N+2}$行: 每行一对整数$\red{(x,y)}$表示对应闭合栅栏的第$\red k$个顶点的坐标。坐标以逆时针顺序给出。整数绝对值不超过$\red{1000}$。

## 输出格式

如果给出的序列不是一个合法的闭合栅栏，那么输出文件只需要输出`“NOFENCE”`。

输出能被看见的栅栏，栅栏用两端的顶点表示，顶点的输出顺序以输入文件中的顺序为准。把栅栏按照最后一个点在输入文件中的顺序排序。如果两条栅栏的最后一个点是一样的，就以它们第一个点的顺序排序。

```c
13
```
