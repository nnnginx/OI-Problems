## 题目描述

"封印大典启动，请出 $\red{Nescafe }$魂珠！"随着圣主 $\red{applepi }$一声令下，圣剑护法 $\red{rainbow}$
和魔杖护法 $\red{freda }$将 $\red{Nescafe }$魂珠放置于封印台上。封印台是一个树形的结构，魂珠放置的
位置就是根节点（编号为 $\red{0}$）。还有 $\red{n }$个其它节点（编号 $\red{1 \sim n}$）上放置着封印石，编号为 $\red{i}$
的封印石需要从魂珠上获取 $\red{E_i }$的能量。能量只能沿着树边从魂珠传向封印石，每条边有一
个能够传递的能量上限 $\red{W_i，}$魂珠的能量是无穷大的。作为封印开始前的准备工作，请你求
出最多能满足多少颗封印石的能量需求？
注意：能量可以经过一个节点，不满足它的需求而传向下一个节点。每条边仅能传递一
次能量。

## 输入格式

第一行一个整数 $\red{n，}$表示除根节点之外其它节点的数量。

接下来 $\red{n }$行，第 $\red{i+1 }$行有三个整数 $\red{F_i}$、$\red{E_i}$、$\red{W_i，}$分别表示 $\red{i }$号节点的父节点、$\red{i }$号节点
上封印石的能量需求、连接节点 $\red{i }$与 $\red{F_i }$的边最多能传递多少能量。

## 输出格式

最多能满足多少颗封印石的能量需求。

## 样例

#### 输入样例

```c
4
0 3 2
0 100 100
1 1 1
2 75 80
```

#### 输出样例

```c
2
```

## 提示

对于 $\red{100\% }$的数据，满足 $\red{1<=n<=1000，}$$\red{0<=F_i<=n，}$$\red{0<=E_i,W_i<=100}$。
