## 题目描述

定义由若干不相交的"$\red{red}$"组成的字符串为"好串"；例如"$\red{redred}$","$\red{rreded}$"是好串，都包含两个不相交的"$\red{red}$"；"$\red{redrde}$"不是好串。由于字符串中有些位置的字符信息丢失了，所以有些位置可能是’$\red{r}$’,’$\red{e}$’,’$\red{d}$’中的任意一个。现在给你一个由’$\red{r}$’,’$\red{e}$’,’$\red{d}$’,’$\red?$’组成的字符串，你能否判断它有没有可能是一个好串呢？

## 输入描述

包含多组输入输出，第一行输入一个整数$\red{T}$表示输入的组数；

每组输入仅包含一行，为字符串$\red{s，}$且只包含字符’$\red{r}$’,’$\red{e}$’,’$\red{d}$’,’$\red?$’.

## 输出描述

对于每一组输入，输出"$\red{Yes}$"或"$\red{No}$"，分别表示可能和不可能。

## 样例

#### 输入样例

```c
6
????dd
???
??d
redder
?e?
r??e?d
```

#### 输出样例

```c
Yes
Yes
Yes
No
Yes
Yes
```

## 提示

对于$\red{100\%}$的数据，$\red{1<=T<=10^5,}$并且$\red{s}$的长度之和不超过$\red{3\times 10^5}$;

对于其中$\red{20\%}$的数据，$\red{s}$的长度之和不超过$\red{50}$；

另有$\red{20\%}$的数据，$\red{s}$的长度之和不超过$\red{300}$；

另有$\red{10\%}$的数据，$\red{s}$的长度均不超过$\red{3}$。
