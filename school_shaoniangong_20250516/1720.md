## 题目描述

邪狼终于理解了高精度数除以低精度数$\red{1}$的程序，于是他开始思考当输入高精度数$\red{a}$（不超过$\red{100}$位）和低精度数$\red{b}$后，如何求包含小数的$\red{a/b}$的值。为了简单起见，他开辟了一个可容纳$\red{101}$个元素的整数数组以保存商，并且由于数组下标为$\red{0}$的位置用于保存结果即$\red{a/b}$的商中的小数点位置，因此实际存放的商的位数最多为$\red{100}$位。

## 输入格式

共两行，第一行为数字字符串$\red{a}$，长度不大于$\red{100}$，第二行为低精度数$\red{b}$。

## 输出格式

其格式为整数加小数（不包含小数点）不超过$\red{100}$位，如整数前有$\red{0}$或小数最末尾有$\red{0}$，则自动舍弃，末尾一数为余数，与前面的结果以$\red{“,”}$分隔。如除数为$\red{0}$，则输出$\red{“Divisor\ is\ 0”}$。

## 样例

#### 输入样例 1

```c
1  5
```

#### 输出样例 1

```c
0.2,0
```

#### 输入样例 2

```c
1000  3333
```

#### 输出样例 2

```c
0.300030003000300030003000300030003000300030003000300030003000300030003000300030003000300030003,1000
```

#### 输入样例 3

```c
1  123456789
```

#### 输出样例 3

```c
0.000000008100000073710000670761006103925155545718915466042130740983389742948846660834504613593991983,87077413
```
