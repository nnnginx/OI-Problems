## 题目描述

在大学期间，经常需要租借教室。大到院系举办活动，小到学习小组自习讨论，都需要向学校申请借教室。教室的大小功能不同，借教室人的身份不同，借教室的手续也不一样。

面对海量租借教室的信息，我们自然希望编程解决这个问题。

我们需要处理接下来 $\red{n}$ 天的借教室信息，其中第 $\red{i}$ 天学校有 $\red{r_i}$ 个教室可供租借。共有 $\red{m}$ 份订单，每份订单用三个正整数描述，分别为 $\red{d_j}$，$\red{s_j}$，$\red{t_j}$，表示某租借者需要从第 $\red{s_j}$ 天到第 $\red{t_j}$ 天租借教室（包括第 $\red{s_j}$ 天和第 $\red{t_j}$ 天），每天需要租借 $\red{d_j}$ 个教室。

我们假定，租借者对教室的大小、地点没有要求。即对于每份订单，我们只需要每天提供 $\red{d_j}$ 个教室，而它们具体是哪些教室，每天是否是相同的教室则不用考虑。

借教室的原则是先到先得，也就是说我们要按照订单的先后顺序依次为每份订单分配教室。如果在分配的过程中遇到一份订单无法完全满足，则需要停止教室的分配，通知当前申请人修改订单。这里的无法满足指从第 $\red{s_j}$ 天到第 $\red{t_j}$ 天中有至少一天剩余的教室数量不足 $\red{d_j}$ 个。

现在我们需要知道，是否会有订单无法完全满足。如果有，需要通知哪一个申请人修改订单。

## 输入格式

第一行包含两个正整数 $\red{n}$，$\red{m}$，表示天数和订单的数量。

第二行包含 $\red{n}$ 个正整数，其中第 $\red{i}$ 个数为 $\red{r_i}$，表示第 $\red{}i$ 天可用于租借的教室数量。

接下来有 $\red{m}$ 行，每行包含三个正整数 $\red{d_j}$，$\red{s_j}$，$\red{t_j}$，表示租借的数量，租借开始、结束分别在第几天。每行相邻的两个数之间均用一个空格隔开。天数与订单均用从 $\red{1}$ 开始的整数编号。

## 输出格式

如果所有订单均可满足，则输出只有一行，包含一个整数 $\red{0}$。

否则（订单无法完全满足）输出两行，第一行输出一个负整数 $\red{-1}$，第二行输出需要修改订单的申请人编号。

## 样例

## 样例输入 $\red1$

```C
4 3
2 5 4 3
2 1 3
3 2 4
4 2 4
```

## 样例输出 1

```C
-1
2
```

## 样例 $\red1$ 解释

第$\red{ 1 }$份订单满足后，$\red{4}$ 天剩余的教室数分别为 $\red{0}$，$\red{3}$，$\red{2}$，$\red{3}$。第 $\red{2}$ 份订单要求第 $\red{2}$ 天到第 $\red{4}$ 天每天提供$\red{ 3 }$个教室，而第 $\red{3}$ 天剩余的教室数为 $\red{2}$，因此无法满足。分配停止，通知第 $\red{2}$ 个申请人修改订单。

## 数据范围与提示

对于 $\red{10\% }$的数据，有 $1\red{ \leq n, m \leq 10^1}$；

对于 $\red{30\% }$的数据，有 $\red{1 \leq n, m \leq 10^3}$；

对于 $\red{70\%}$ 的数据，有 $\red{1 \leq n, m \leq 10^5}$；

对于 $\red{100\%}$ 的数据，有 $\red{1 \leq n, m \leq 10^6}$，$\red{0 \leq r_i, d_j \leq 10^9}$，$\red{1 \leq s_j \leq t_j \leq n}$。
