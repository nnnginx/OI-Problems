

# Description

不用上课的时候，huhe总是喜欢去玩密室逃脱，学编程的他对此十分拿手，总是用各种算法破解别人的地图，密室的老板们对他恨之入骨，但又无可奈何。

这一天，老板们联合起来，要给huhe一个刻骨铭心的教训，成功地把他困在了某个地方...

可是huhe教了很多学生，其中一个就是著名的FGQ，FGQ拿到了一张坐标图，他飞奔去营救恩师huhe，s为FGQ的初始位置，m为huhe的位置，‘.’为安全通道，FGQ可以在上面跑，每单位时间可以移动一格；‘#’为半安全通道，FGQ在上面得仔细观察再通过，每两单位时间可以移动一格；‘o’是障碍物，FGQ不能在它上面行动。也就是说，FGQ只能在安全或半安全通道上行走，必须绕过障碍物，并到达huhe所在的地点。但是愤怒的老板们看见这么多人来就huhe，可能会很生气，进而对huhe不利，请FGQ在黄金营救时间t内解决问题

# Format

## Input

依次输入黄金营救时间t(0<t<1000)，坐标图的长x,宽y(5<=x,y<=25)，和整张坐标图。

## Output

判断你的最优方案是否可以在黄金营救时间内救出huhe，如赶到，输出所用时间；如未赶到，输出huhe的哭声――“55555”（不包括引号）。

# Samples

```input1
11
10
8
......s...
..........
#ooooooo.o
#.........
#.........
#.........
#.....m...
#.........
```

```output1
10
```

# Limitation

1s, 1024KiB for each test case.
