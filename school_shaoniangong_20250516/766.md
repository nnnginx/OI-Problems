## 题目描述

图书馆中每本书都有一个图书编码，可以用于快速检索图书，这个图书编码是一个 正整数。 每位借书的读者手中有一个需求码，这个需求码也是一个正整数。如果一本书的图 书编码恰好以读者的需求码结尾，那么这本书就是这位读者所需要的。 小 D 刚刚当上图书馆的管理员，她知道图书馆里所有书的图书编码，她请你帮她写 一个程序，对于每一位读者，求出他所需要的书中图书编码最小的那本书，如果没有他 需要的书，请输出$\red{-1}$。

## 输入格式

输入文件的第一行，包含两个正整数 $\red{n}$ 和 $\red{q}$，以一个空格分开，分别代表图书馆里 书的数量和读者的数量。接下来的 $\red{n}$ 行，每行包含一个正整数，代表图书馆里某本书的图书编码。接下来的 $\red{q}$ 行，每行包含两个正整数，以一个空格分开，第一个正整数代表图书馆 里读者的需求码的长度，第二个正整数代表读者的需求码。

## 输出格式

输出文件有 $\red{q}$ 行，每行包含一个整数，如果存在第 $\red{i}$ 个读者所需要的书，则在第 $\red{i}$ 行输出第 $\red{i}$ 个读者所需要的书中图书编码最小的那本书的图书编码，否则输出$\red{-1}$。

## 样例

### 样例输入1

```plain
5 5 
2123 
1123 
23 
24 
24 
2 23 
3 123 
3 124 
2 12 
2 12
```

### 样例输出1

```plain
23
1123
-1
-1
-1
```

## 数据范围与提示

对于 $\red{20\%}$的数据，$\red{1 \le n \le}$$\red{2}$。另有 $\red{20\%}$的数据，$\red{q = 1}$。另有 $\red{20\%}$的数据，所有读者的需求码的长度均为 $\red{1}$。另有 $\red{20\%}$的数据，所有的图书编码按从小到大的顺序给出。对于 $\red{100\%}$的数据，$\red{1 \leq n \leq 1,000，1 \leq q \leq 1,000}$，所有的图书编码和需求码均不超过 $\red{10,000,000}$。
