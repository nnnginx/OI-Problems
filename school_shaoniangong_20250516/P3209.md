
## 题目背景

在 WX 上用户 $a$ 想要找用户 $b$，但他们之间不一定有好友关系，这就需要一个中间人 $c$ 来帮助他们了。

## 题目描述

有 $n$ 个用户，每个用户 $x$ 时间到 $y$ 时间在线。

有 $m$ 个原始关系 $ab$，表示开始时 $ab$ 就已经是好友。（这个关系是双向的）

加下来有 $q$ 次询问，每次询问输入 $a、b、t$,表示 $a$ 想在 $t$ 时间和 $b$ 通讯。如果 $a$ 和 $b$ 已经是好友，那就可以直接通通讯，除此之外，还有 $2$ 种可能可以给 $b$ 通讯：

1.如果 $a$ 有 $c$ 的好友且 $c$ 有 $b$ 好友且 $c$ 此时在线那么 $a$就可以向 $c$ 要 $b$ 的好友并通讯。（求助的时间忽略不计算，如果 $a$ 要到了 $b$ 的好友那下一次通讯就可以直接通讯了）

2.有 $s$ 次机会可以使用小X研发的魔法石使得 $a$ 和 $b$ 直接通讯一次（这个通讯是一次性的，也就是说这一次通讯了下一次 $a$ 的好友里并没有 $b$，不能直接通讯）。

如果 $a$ 最后可以和 $b$ 通讯，输出 YES，否则输出 NO（就算 $a$ 给 $b$ 通讯时 $b$ 不在线也视为通讯成功[让我们赞美良心的出题人吧！！！（bushi）]）。

## 输入格式

第一行 $3$ 个整数 $n、m、s$。

先输入 $n$ 行每个用户的在线时间，再输入 $m$ 行表示原始关系。

接下来输入 $q$。

然后输入 $q$ 行，每行 $3$ 个数，$a、b、t$,表示 $a$ 想在 $t$ 时间和 $b$ 通讯。

## 输出格式

一共 $q$ 行，代表每一次询问的结果。

## 样例 #1

### 样例输入 #1

```
3 2 1
10:00 11:00
10:30 11:20
9:00 11:00
1 2
2 3
1
1 3 10:00
```

### 样例输出 #1

```
YES
```

## 提示

### 样例解释

用户 $1$ 想在 `10:00` 给用户 $3$ 通讯，但 $1$ 和 $3$ 不是好友，所以不能直接通讯，但是此时的 $2$ 还未上线所以无法向 $2$ 索要 $3$ 的好友，但在样例中，有一块魔法石，所以直接使用魔法石通讯。

温馨提示：多组询问！！！

对应 $10\%$ 的数据，$1 \le n,m,q  \le 5$

对应 $100\%$ 的数据，$1 \le n,m,q \le 2 \times 10^5$

题目保证时间在 `00:00` 至 `23:59` 的范围内。
