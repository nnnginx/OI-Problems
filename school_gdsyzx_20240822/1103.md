# Description

交换二进制数相邻两个位置的数字，需要花费 1 元的代价。

读入整数 n 以及 n 位二进制数（也许有前导 0），你需要依次回答 n 个独立的问题，第 i 个问题（1 ≤ i ≤ n）是这样的：

假如要使得读入的二进制数是 $2^i$ 的倍数，至少需要花费多少元的代价？如果不可能，则输出 -1。

注意：每个问题都是独立的，不会相互影响，也就是说，每个问题只是假设，不会真的改变 n 位二进制数，纯粹是为了计算答案而已。

# Format

## Input

第一行，一个整数 n。1 ≤ n ≤ 100000。

第二行，n 位二进制数，每位是 0 或 1。

## Output

一行，共 n 个整数。

# Samples

```input1
5
10101
```

```output1
1 3 -1 -1 -1
```

# Limitation

对于 60% 的数据，n ≤ 10。
