# Problem A. 六六大顺

## 时空限制

1s / 128MB

## 题目描述

一天，橘子老师得到了 $1$ 个橘子，重量是 $666$克。橘子老师对此非常满意，因为这个橘子的重量刚好是由 $3$ 个 $6$ 组成的 “$666$”，这个数字听起来十分的顺，毕竟”六六大顺“。但并不是每个橘子都有这么完美的重量！现在给你一个橘子的重量，请你帮橘子老师估计在橘子老师得到这个橘子后的满意程度。

具体来说，给你一个正整数 $n$，代表橘子的重量，如果数字 $n$ 中含有的 $6$ 的个数为 $m$：

当 $m≥3$，橘子老师将非常满意，输出 “$666$”；

当 $m = 2$，橘子老师比较满意，输出 “$66$”；

当 $m < 2$，橘子老师将非常不满意，输出 “$555$”。

## 输入格式

输入一个正整数 $n$ （$0 < n < 100000$）;

## 输出格式

输出一行字符串，表示橘子老师的满意程度 ("$666$", "$66$", "$555$"中的一个)

## 输入样例1：

```input
6666
```

## 输出样例1：

```output
666
```

## 输入样例2：

```input
166
```

## 输出样例2：

```output
66
```

## 输入样例3：

```input
100
```

## 输出样例3：

```output
555
```
