## 问题描述

话说这天，老冯在刷《鸡汤来咯》。

老冯发现他看 $B$ 站分辨率极低，他就在想为什么这么低。

视频传输时是把每一帧的图片传输过来，实际上就相当于传输了一个表示图片的数组，肯定是数组太小了，所以图片就模糊了，进而导致视频分辨率低。

于是他决定计算一个 $n$ $MB$，每个元素都是 $32$ 位二进制整数的数组能存储多少个元素，来验证自己的猜想是不是正确的。

## 输入格式

一个整数 $n$ 。

## 输出格式

一个整数，即数组最多能存储多少个元素。

```input1
434
```

```output1
113770496
```

## 数据范围

$1 \leqslant n \leqslant 2048$
