## 问题描述

一个优秀的人工智能一定会模拟退火吧。

变体精灵很厉害，可以将它的力量与敏捷属性相互转化，但每秒要消耗魔法值。

现在，在 *阿哈利姆的天地迷宫* 中，阿哈利姆对变体精灵进行了强化，使得它可以让力量、敏捷与智力属性都可以相互转化、多次转化，而且不再需要消耗魔法值，~~然后成为六边形战士~~。

但作为代价，存在转化率 $c_{ij}(0\leq c_{ij}\leq 1)$ 表示将第 $i$ 种属性转化成第 $j$ 种属性的利用率，即一点的第 $i$ 种属性可以转化成 $c_{ij}$ 点的第 $j$ 种属性。转化率是单向的，不能用 $c_{ij}$ 点 $j$ 属性换成一点 $i$ 属性。

由于变体精灵喜欢上了微操，于是它可以转化任意正实数点的属性成别的属性。

在攻略 *阿哈利姆的天地迷宫* 的过程中，由于它开了一个牛逼的脚本，若变体精灵的第 $i$ 种属性为 $a_i$ ，则可以节省变体精灵 $\frac{a_i}{a_i+777}\times4396 s$ 的时间。

现在，变体精灵很好奇，它最多可以节省多少的时间。

## 输入格式

有多组测试数据。

第一行一个整数 $T$，表示有 $T$ 组数据。
每组测试数据有四行，第一行三个整数，表示变体精灵的原始 *力量值*、*敏捷值*、*智力值*。

接下来一个 $3\times3$的矩阵，第 $i$ 行第 $j$ 列的数 $c_{ij}$ 表示第 $i$ 种属性到第 $j$ 种属性之间的转化率。

## 输出格式

$T$ 行每行一个实数，表示最多节省的时间，保留 $5$ 位小数，与答案误差小于 $10^{-2}$ 时正确。

```input1
1
5 10 15
0.99 0.99 0.99
0.99 0.99 0.99
0.99 0.99 0.99
```

```output1
167.35073
```

```input2
1
5 10 15
1 1 1
1 1 1
1 1 1
```

```output2
167.57306
```

```input3
1
5 10 15
0 0 0
0 0 0
0 0 0
```

```output3
167.22268
```

```input4
1
1 14 11
0.99991 0.99990 1.00000
0.99996 1.00000 0.99292
1.00000 0.95961 0.99907
```

```output4
145.47527
```

```input5
1
19 24 17
0 1 0
1 0 0
0 0 0
```

```output5
330.84977
```

## 数据范围

对于 $10$% 的数据，$c_{ij}=0$ 。

对于另外 $10$% 的数据，$c_{ij}= 1$ 。

对于另外 $10$% 的数据，原始 *力量值* $+$*敏捷值*$ +$*智力值* $\le50$，$T=1$ 。

对于 $100$% 的数据，$0\le c_{ij}\le1$，$1\le$ 原始 *力量值*、*敏捷值*、*智力值* $\le 10^5$，$T\le 10$ 。
