## 问题描述

奶牛Bessie在她的村子里上计算机课，对进制转换比较着迷。

一个 $B$ 进制的整数，从右向左各个数位上的权分别为 $B^0,B^1, B^2, B^3,……$，例如我们所熟悉的十进制，就是 $1, 10, 100, 1000$ 等等。以十进制 $1234$ 为例，就表示 $1(1000) + 2(100) + 3(10) + 4(1)$。如果用五进制表示则为 $1(125)+ 2(25) + 3(5) + 4(1)$，对应十进制中的数就是 $194$。

Bessie注意到对于相同的数，进制越高则数值越大，如 $7$ 进制 $1234$ 大于 $6$ 进制 $1234$。

对于一个 $B$ 进制的整数，每个数位上的数范围为 $0..B-1$， 例如十进制数的范围为 $0..9$；五进制数的范围为 $0..4$。其实我们还应考虑大于 $10$ 的进制情况。计算机科学家们通常采用的是 $16$ 进制 ("hexadecimal")，用 $A..F$ 表示 $10..15$。例如 $16$ 进制 $BEEF$ 则为 $11(4096) +14(256) + 14(16) + 15(1)$，结果为 $10$ 进制数 $48879$。

Bessie对大于 $10$ 的进制很感兴趣。她想出一个整数  $N$，然后用两个不同进制 $X$ 和 $Y$ 写出。有趣的是，这两个不同进制写出的数都是 $3$ 位数，且每个数位上的数都是数字，范围为 $1..9$。不幸的是，由于Bessie可怜的记忆，她已经忘记了 $N，X，Y$。目前只有她写下的两个整数，请帮她推算出进制 $X$ 和 $Y$ 。

注意如果使用蛮力的方法，则运算量为 $15000^2$，肯定超时，将得不到满分。

## 输入格式

第一行包含一个整数 $K$，表示接下来有 $K$ 对进制转换后的数。

接下来 $K$ 行，每行两个三位数，中间用一个空格隔开，分别表示 $N$ 用 $X$ 和 $Y$ 进制转换后的数。

## 输出格式

输出共有 $K$ 行，每行包含两个整数，中间用一个空格隔开，分别表示 $X$ 和 $Y$ 的值，输入保证有解。

```input1
1
419 792
```

```output1
47 35
```

## 样例解释

十进制数 $8892$ 在 $47$进制下是 $419$，在 $35$ 进制下是 $792$

## 数据f范围

对于 $30\%$ 的数据：$1≤K≤30$；

对于 $50\%$ 的数据：$1≤K≤50$；

对于 $100\%$ 的数据：$1≤K≤100；10≤X,Y≤15,000$；
