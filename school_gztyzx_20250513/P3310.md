## 问题描述

自从 D 国的情报中心被 C 国击溃后，D 国长期处于危机状态。而如今，D 国只剩下51区的情报中心没有恢复。小 P 作为 D 国的高级工程师，他接受了这个任务。

D 国51区曾是 D 国的军事要塞，这里地势平坦，一望无际。因此，我们可以将 D 国51区抽象成一个无限大的二维平面直角坐标系。规定51区中心的的坐标为 $(0,0)$，正北方为 $x$ 轴的正方向，正东方为 $y$ 轴的正方向。平面坐标系的单位长度为 $1m$。

51区分布着 $n$ 个情报机器人，它们是51区的情报的重要来源。机器人有两种通信方式，一种是通过机器人存储的相应机器人的信息直接通信，一种是通过机器人内置的信号通信，若机器人与另一个机器人的信号通信范围有重合的部分，两个机器人可以互相通信。

由于当地情报中心被黑客攻击，所有的机器人也丢失了大部分数据，其中，机器人的通信信息全部丢失，信号装置也受损严重。

小 P 的第一步就是恢复机器人的通信功能。

小 P 通过电脑查询到了 $n$ 个机器人的坐标 $(x_i,y_i)$ ，以及每个机器人固定的行走方向 $d_i$。等到 $t=0$ 时刻时，小 P 会同时按下所有机器人的启动按钮，所有机器人将会同时启动。机器人的信号通信范围会逐渐恢复扩大。具体来说，$t=0$ 时只有当前机器人所在位置在信号范围内，当 $t\geq1$ 时，范围会在 $t-1$ 的基础上多出所有与 $t-1$ 时刻信号通信范围内的点相邻的，且之前不在 $t-1$ 时刻范围内的点（均相对于当前机器人的位置），你可以根据图1更好的了解这个约定：

![](/upload/image/20201030/20201030133029_15407.png)

如图1，红，橙，黄，分别表示 $0，1，2$ 时刻新增加的点，为了更加直观，我们将一个点化为了一个方格。

每一时刻，机器人会向规定方向移动一次，然后同时向信号范围发送和接收信息，如果接收到了之前从未记录过机器人的信息，那么机器人会向之前已经记录过的机器人都发送一个该机器人的信息，并存储这个机器人的信息。以上操作都是同时发生的，接受信息和发送信息不分先后。机器人在 $t$ 时刻发送的信息会在 $t+1$ 时刻消失。

值得注意的是，由于机器人都十分微小，你可以认为两个机器人在同一个位置时不会产生碰撞。

由于51区实在是太大了，机器人运动的速度又很慢，他现在想知道他按下按键之后，最早会在哪个时刻使得每个机器人都记录着所有 $n$ 个机器人的信息，以方便他知道该怎么设闹钟。

与 $(x,y)$ 相邻的点包括 $(x-1,y) , (x+1,y) , (x,y-1) , \left(x,y+1\right)$。

设当前点为 $(x,y)$ ，那么方向 $d_i$ 与对应的移动一步的到达的位置规定如下:

| di=  | 到达位置  |
| ---- | --------- |
| 1    | (x+1,y)   |
| 2    | (x-1,y)   |
| 3    | (x,y+1)   |
| 4    | (x,y-1)   |
| 5    | (x+1,y+1) |
| 6    | (x+1,y-1) |
| 7    | (x-1,y+1) |
| 8    | (x-1,y-1) |

## 输入格式

第一行一个整数 $n$，表示机器人的数量。

接下来 $n$ 行每行三个整数 $x_i , y_i , d_i$ ，含义见【题目描述】。

## 输出格式

仅一行，表示最小时间，如果无论怎么样都不可能完成，输出 `No solutions`。

```input1
3
0 -1 3
2 1 4
4 -1 3
```

```output1
1
```

### 样例 1 解释

如图2所示，$t=1$ 时刻时，机器人的信号范围将会互相重合。

红，橙，黄分别为三个机器人的信号范围，绿色为信号范围重合的点。

![](/upload/image/20201030/20201030133720_43878.png)

```input2
3
0 -1 8
2 1 4
4 -1 3
```

```output2
4
```
