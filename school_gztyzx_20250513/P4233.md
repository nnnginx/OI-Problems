# 【题目背景】

这天，妈妈叫小明去超市帮忙买瓶酱油。

# 【题目描述】

小明来到超市，买了单。还剩了许多硬币。小明看着面前杂乱无章的硬币，强迫症又犯了。

小明想要这些硬币排列成他想要的样子。可是小明太笨拙了。他每次翻转硬币时都会把相邻的硬币翻转过来（可以是左边，也可以是右边，对于两端的硬币，可以选择翻转一个或两个）。

小明想知道要翻转多少次才能把硬币翻转成他想要的样子。但是小明太笨了。所以他就找到了会编程的你。帮助他解决这个问题。

# 【输入格式】

共两行。

第一行一个正整数 $n$ 表示硬币的个数。

第一行数组 $a$， 由 $1$ 和 $0$ 组成的字符串（$1$ 表示正面，$0$ 表示反面），表示硬币的初始状态。

第二行同样一个数组 $b$，由 $1$ 和 $0$ 组成的字符串，表示小明希望硬币排成的样子。

# 【输出格式】

一个数字 $n$，表示小明要将硬币翻转到目标状态最少需要的翻转数。

```input1
5
1 1 1 0 0
0 0 1 1 0
```

```output1
3
```

# 【样例解释】

先将第 $1$ 个硬币翻转，将第 $2$ 个硬币带着翻转。再将第 $4$ 个硬币翻转，带动第 $5$ 个硬币。最后将第 $5$ 个硬币翻转。

共用 $3$ 步。

# 数据范围

对于 $20\%$ 的数据，保证 $1 \leq n \leq 50$

对于 $100\%$ 的数据，保证 $1 \leq n \leq 500000$
