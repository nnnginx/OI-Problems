## 题目描述

主机（`Server`）和PC（`Client`）。主机负责建立连接，PC负责加入连接。

需要进行网络连接的计算机共有 $n$ 台，编号为 $1 \sim n$，这些机器将按编号递增的顺序，依次发起一条建立连接或加入连接的操作。

建立和加入必须符合规范：

1. 必须形如 `a.b.c.d:e` 的格式，其中 $a, b, c, d, e$ 均为非负整数；
2. $0 \le a, b, c, d \le 255$，$0 \le e \le 65535$；
3. $a, b, c, d, e$ 均不能含有多余的前导 $0$。

相应地，不符合规范的格式串可能具有以下特征：

1. 不是形如 `a.b.c.d:e` 格式的字符串，例如含有多于 $3$ 个字符 `.` 或多于 $1$ 个字符 `:` 等情况；
2. 整数 $a, b, c, d, e$ 中某一个或多个超出上述范围；
3. 整数 $a, b, c, d, e$ 中某一个或多个含有多余的前导 $0$。

例如，格式串 `192.168.0.255:80` 是符合规范的，但 `192.168.0.999:80`、`192.168.00.1:10`、`192.168.0.1:088`、`192:168:0:1.233` 均是不符合规范的。

如果主机或PC在发起操作时提供的格式串不符合规范，这条操作将被直接忽略。

在本问题中，我们假定凡是符合上述规范的格式串均可参与正常的连接，你无需考虑每个格式串的实际意义。

由于网络阻塞等原因，不允许两台主机使用相同的格式串，如果此类现象发生，后一台尝试建立连接的主机将会无法成功建立连接；除此之外，凡是提供符合规范的格式串的主机均可成功建立连接。

如果某台提供符合规范的地址的客户机在尝试加入连接时，与先前某台已经成功建立连接的主机提供的格式串相同，这台客户机就可以成功加入连接，并称其连接到这台主机；如果找不到这样的主机，则认为这台客户机无法成功加入连接。

请注意，尽管不允许两台不同的主机使用相同的格式串，但多台客户机使用同样的格式串，以及同一台主机同时被多台客户机连接的情况是被允许的。

你的任务很简单：在给出每台计算机的类型以及格式串之后，判断这台计算机的连接情况。

## 输入格式

第一行，一个正整数 $n$。

接下来 $n$ 行，每行两个字符串 $\mathit{op}, \mathit{ad}$，按照编号从小到大给出每台计算机的类型及格式串。

其中 $\mathit{op}$ 保证为字符串 `Server` 或 `Client` 之一，$\mathit{ad}$ 为一个长度不超过 $25$ 的，仅由数字、字符 `.` 和字符 `:` 组成的非空字符串。

每行的两个字符串之间用恰好一个空格分隔开，每行的末尾没有多余的空格。

## 输出格式

输出共 $n$ 行，每行一个正整数或字符串表示第 $i$ 台计算机的连接状态。其中：

如果第 $i$ 台计算机为主机，则：

1. 如果其提供符合规范的格式串且成功建立连接，输出字符串 `OK`。
2. 如果其提供符合规范的格式串，但由于先前有相同格式串的主机而无法成功建立连接，输出字符串 `FAIL`。
3. 如果其提供的格式串不是符合规范的格式串，输出字符串 `ERR`。

如果第 $i$ 台计算机为客户机，则：

1. 如果其提供符合规范的格式串且成功加入连接，输出一个正整数表示这台客户机连接到的主机的编号。
2. 如果其提供符合规范的格式串，但无法成功加入连接时，输出字符串 `FAIL`。
3. 如果其提供的格式串不是符合规范的格式串，输出字符串 `ERR`。

## 样例 #1

### 样例输入 #1

```
5

Server 192.168.1.1:8080

Server 192.168.1.1:8080

Client 192.168.1.1:8080

Client 192.168.1.1:80

Client 192.168.1.1:99999
```

### 样例输出 #1

```
OK

FAIL

1

FAIL

ERR
```

## 提示

**【样例解释 #1】**

计算机 $1$ 为主机，提供符合规范的格式串 `192.168.1.1:8080`，成功建立连接；

计算机 $2$ 为主机，提供与计算机 $1$ 相同的格式串，未能成功建立连接；

计算机 $3$ 为客户机，提供符合规范的格式串 `192.168.1.1:8080`，成功加入连接，并连接到主机 $1$；

计算机 $4$ 为客户机，提供符合规范的格式串 `192.168.1.1:80`，找不到主机与其连接；

计算机 $5$ 为客户机，提供的格式串 `192.168.1.1:99999` 不符合规范。

**【数据范围】**

| 测试点编号 | $n \le$ | 特殊性质 |

|:-:|:-:|:-:|

| $1$ | $10$ | 性质 1 2 3 |

| $2 \sim 3$ | $100$ | 性质 1 2 3 |

| $4 \sim 5$ | $1000$ | 性质 1 2 3 |

| $6 \sim 8$ | $1000$ | 性质 1 2 |

| $9 \sim 11$ | $1000$ | 性质 1 |

| $12 \sim 13$ | $1000$ | 性质 2 |

| $14 \sim 15$ | $1000$ | 性质 4 |

| $16 \sim 17$ | $1000$ | 性质 5 |

| $18 \sim 20$ | $1000$ | 无特殊性质 |

“性质 1”为：保证所有的格式串均符合规范；

“性质 2”为：保证对于任意两台不同的计算机，如果它们同为主机或者同为客户机，则它们提供的格式串一定不同；

“性质 3”为：保证任意一台主机的编号都小于所有的客户机；

“性质 4”为：保证所有的格式串均形如 `a.b.c.d:e` 的格式，其中 $a, b, c, d, e$ 均为不超过 ${10}^9$ 且不含有多余前导 $0$ 的非负整数；

“性质 5”为：保证所有的格式串均形如 `a.b.c.d:e` 的格式，其中 $a, b, c, d, e$ 均为只含有数字的非空字符串。

对于 $100 \%$ 的数据，保证 $1 \le n \le 1000$。
