## 问题陈述

高桥和青木喜欢计算，所以他们现在要玩数字。

首先，他们各自想出了一个正整数。高桥想出了 $X$，青木想出了 $Y$。
然后，他们将通过重复以下操作 $K$ 次来享受乐趣：

- 计算高桥当前保持的数字和青木当前保持的数字的按位与。设 $Z$ 为结果。
- 然后，将 $Z$ 加到高桥和青木保持的两个数字上。

然而，对于这两个数学狂热者来说，这实在是太麻烦了。
你能找出高桥最终保持的数字和青木最终保持的数字吗？

请注意，输入和输出都是以二进制进行的。
特别地，$X$ 和 $Y$ 作为长度为 $N$ 和 $M$ 的字符串 $S$ 和 $T$ 给出，由 `0` 和 `1` 组成，且其初始字符保证为 `1`。

## 约束条件

- $1 \leq K \leq 10^6$
- $1 \leq N,M \leq 10^6$
- 字符串 $S$ 和 $T$ 的初始字符为 `1`。

## 输入

输入通过标准输入以以下格式给出：

> $N$ $M$ $K$
> 
> $S$
> 
> $T$

## 输出

在第一行，打印高桥最终会保持的数字；在第二行，打印青木最终会保持的数字。
这两个数字应以二进制形式表示，并打印为以 `1` 开头的 `0` 和 `1` 组成的字符串。

```input1
2 3 3
11
101
```

```output1
10000
10010
```

每次操作后 $X$ 和 $Y$ 的值如下：

- 第一次操作后：$(X,Y)=(4,6)$。
- 第二次操作后：$(X,Y)=(8,10)$。
- 第三次操作后：$(X,Y)=(16,18)$。

```input2
5 8 3
10101
10101001
```

```output2
100000
10110100
```

```input3
10 10 10
1100110011
1011001101
```

```output3
10000100000010001000
10000100000000100010
```
