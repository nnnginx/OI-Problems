## Description

众所周知，OJ系统每年会不定期更新，但是OJ团队一共有n位成员，管理员希望他们都能同时收到OJ站内发送的系统升级信息，且系统升级完成会自动发送消息。但是站内信息一小时会统一发送一次，并且是整点发送，但是为了保证成员的休息时间，站内信在仅可以在早8时30分至晚21时发送信息，如果系统升级完成晚于或等于21时，则消息在第二天8点统一发送。

现在管理员希望知道系统升级后多长时间，OJ团队成员能够收到站内消息。

## Input Format

输入仅有一行
输入两个数h,m，分别代表系统升级完成的小时和分钟；

## Output Format

输出仅一个数字，代表系统升级成功后至团队成员收到站内信的**分钟数**（将小时数化为分钟数）

如果输入格式不符合规范，则输出大写字母 X

```input1
18 7
```
```output1
53
```
```input2
21 0
```
```output2
690
```
```input3
15 0
```
```output3
0
```
