<h2>Description</h2><span lang="en-us"><p>Assume you have a square of size <i>n</i> that is divided into <i>n × n</i> positions just as a checkerboard. Two positions (<i>x</i><sub>1</sub>, <i>y</i><sub>1</sub>) and (<i>x</i><sub>2</sub>, <i>y</i><sub>2</sub>), where 1 ≤ <i>x</i><sub>1</sub>, <i>y</i><sub>1</sub>, <i>x</i><sub>2</sub>, <i>y</i><sub>2</sub> ≤ <i>n</i>, are called “independent” if they occupy different rows and different columns, that is, <i>x</i><sub>1</sub> ≠ <i>x</i><sub>2</sub> and <i>y</i><sub>1</sub> ≠ <i>y</i><sub>2</sub>. More generally, <i>n</i> positions are called independent if they are pairwise independent. It follows that there are <i>n</i>! different ways to choose <i>n</i> independent positions. </p><p>Assume further that a number is written in each position of such an <i>n</i> × <i>n</i> square. This square is called “homogeneous” if the sum of the numbers written in <i>n</i> independent positions is the same, no matter how the positions are chosen. Write a program to determine if a given square is homogeneous!</p></span><h2>Input</h2><span lang="en-us"><p>The input contains several test cases.</p><p>The first line of each test case contains an integer <i>n</i> (1 ≤ <i>n</i> ≤ 1000). Each of the next <i>n</i> lines contains <i>n</i> numbers, separated by exactly one space character. Each number is an integer from the interval [−1000000, 1000000].</p><p>The last test case is followed by a zero.</p></span><h2>Output</h2><p>For each test case output whether the specified square is homogeneous or not. Adhere to the format shown in the sample output.</p><pre><code class="language-input1">2
1 2
3 4
3
1 3 4
8 6 -2
-3 4 0
0</code></pre><pre><code class="language-output1">homogeneous
not homogeneous</code></pre><h2>Source</h2><a href="searchproblem?field=source&amp;key=Ulm+Local+2006">Ulm Local 2006</a>
