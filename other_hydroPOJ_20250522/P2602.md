<h2>Description</h2><p>The creators of a new programming language D++ have found out that whatever limit for SuperLongInt type they make, sometimes programmers need to operate even larger numbers. A limit of 1000 digits is so small... You have to find the sum of two numbers with maximal size of 1.000.000 digits.</p><h2>Input</h2><p>The first line of an input file contains a single number N (1&lt;=N&lt;=1000000) - the length of the integers (in order to make their lengths equal, some leading zeroes can be added). It is followed by these integers written in columns. That is, the next N lines contain two digits each, divided by a space. Each of the two given integers is not less than 1, and the length of their sum does not exceed N. </p><h2>Output</h2><p>Output file should contain exactly N digits in a single line representing the sum of these two integers.</p><pre><code class="language-input1">4
0 4
4 2
6 8
3 7
</code></pre><pre><code class="language-output1">4750</code></pre><h2>Hint</h2><p>Huge input,scanf is recommended.</p><h2>Source</h2><a href="searchproblem?field=source&amp;key=Ural+State+University+collegiate+programming+contest+2000">Ural State University collegiate programming contest 2000</a>
