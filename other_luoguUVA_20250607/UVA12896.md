# UVA12896 Mobile SMS

## 题目描述

Ali 被认为非常吝啬。他买了一部二手旧手机。他通常用手机接听别人的电话，很少打电话。突然间，他发现发送短信比打电话便宜得多。因此，他总是发送短信而不是打电话。目前，他的手机屏幕出现了问题，显示器上什么都看不到。这使得他很难写短信。但他知道手机键盘的按键排列。现在他请求你的帮助来找出他在写短信时输入的文本。他会告诉你按了哪些键以及按了多少次。手机键盘的排列如下图所示，其中 `_` 表示空格。

![](https://cdn.luogu.com.cn/upload/image_hosting/7uc41ks5.png)

## 输入格式

输入文件包含多个测试用例。输入文件的第一行包含一个整数 $T < 1000$，表示测试用例的数量。接下来是 $T$ 个测试用例。每个测试用例以一个正整数 $5 \leq L \leq 100$ 开始，表示消息的长度。接下来的两行中每行包含 $L$ 个正整数。第一行包含 $0 \leq N _ i \leq 9$（$1 \leq i \leq L$），第二行包含 $1 \leq P_i \leq 4$（$1 \leq i \leq L$）。$N _ i$ 是他按的键盘数字，$P _ i$ 表示他按下 $N _ i$ 键的次数。

## 输出格式

对于每组输入，输出一行消息，表示预期的短信内容。参考样例输入输出以获得进一步的说明。
