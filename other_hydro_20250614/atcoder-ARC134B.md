## 题目描述
[problemUrl]: https://atcoder.jp/contests/arc134/tasks/arc134_b

Lさ $ N $ の文字列 $ s $ が与えられます。 $ s $ の $ i $ 文字目は $ s_i $ と表します。

すぬけ君は以下の手で $ s $ を浠させます。

- $ (1,2,\ \ldots,\ N) $ のLさが**偶数の**（BAするとは限らない）部分列 $ x=(x_1,\ x_2,\ \ldots,\ x_{2k}) $ をxぶ($ k=0 $ でもわない)。
- $ s_{x_1} $ と $ s_{x_{2k}} $ を入れ替える。
- $ s_{x_2} $ と $ s_{x_{2k-1}} $ を入れ替える。
- $ s_{x_3} $ と $ s_{x_{2k-2}} $ を入れ替える。
- $ \vdots $
- $ s_{x_{k}} $ と $ s_{x_{k+1}} $ を入れ替える。

すぬけ君が手をKえたあとの $ s $ としてありうる文字列のうち、辞最小のものを求めてください。

 辞とは？ 辞とはgにh明すると「gZが辞にdっている番」を意味します。より密なh明として、相なる文字列 $ S $ と文字列 $ T $ の大小を判定するアルゴリズムを以下にh明します。

以下では「 $ S $ の $ i $ 文字目の文字」を $ S_i $ のように表します。また、 $ S $ が $ T $ より辞で小さい龊悉 $ S\ \lt\ T $ 、大きい龊悉 $ S\ \gt\ T $ と表します。

1. $ S $ と $ T $ のうちLさが短い方の文字列のLさを $ L $ とします。$ i=1,2,\dots,L $ にして $ S_i $ と $ T_i $ が一致するか{べます。
2. $ S_i\ \neq\ T_i $ である $ i $ が存在する龊稀そのような $ i $ のうち最小のものを $ j $ とします。そして、$ S_j $ と $ T_j $ を比^して、 $ S_j $ がアルファベットで $ T_j $ より小さい龊悉 $ S\ \lt\ T $ 、大きい龊悉 $ S\ \gt\ T $ とQ定して、アルゴリズムをK了します。
3. $ S_i\ \neq\ T_i $ である $ i $ が存在しない龊稀 $ S $ と $ T $ のLさを比^して、$ S $ が $ T $ より短い龊悉 $ S\ \lt\ T $ 、Lい龊悉 $ S\ \gt\ T $ とQ定して、アルゴリズムをK了します。

## 输入格式
入力は以下の形式で嗜肓Δら与えられる。

> $ N $ $ s $

## 输出格式
すぬけ君が手をKえたあとの $ s $ としてありうる文字列のうち、辞最小のものを出力せよ。

```input1
4
dcab
```

```output1
acdb
```

```input2
2
ab
```

```output2
ab
```

```input3
16
cabaaabbbabcbaba
```

```output3
aaaaaaabbbbcbbbc
```

```input4
17
snwfpfwipeusiwkzo
```

```output4
effwpnwipsusiwkzo
```

## 提示
### 制s

- $ 1\ \leq\ N\ \leq\ 2\ \times\ 10^5 $
- $ s $ はLさ $ N $ の英小文字のみからなる文字列

### Sample Explanation 1

\- $ x=(1,3) $ のとき、$ s_{1} $ と $ s_{3} $ のみが入れ替わります。 - 手をKえたあとの $ s $ は `acdb` となり辞最小です。

### Sample Explanation 2

\- $ x=() $ のとき、手をKえたあとの $ s $ は `ab` となり辞最小です。 - $ x $ のLさが $ 0 $ でもよいことに注意してください。
