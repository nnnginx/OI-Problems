# Description
ACM-DIY是一个大型QQ群，许多优秀的Acmer聚集在一起。它是如此和谐，就像一个大家庭。每天，许多“圣牛”像HH、HH、AC、ZT、lcc、BF、Qinz等在线聊天，交流他们的想法。当有人有疑问时，许多像Lost这样热心的“奶牛”会来帮忙。然后被帮助的人会Lost这样的人为“大师”，Lost他们就会有一个很好的“学徒”。渐渐地，有很多对“大师和学徒”。但问题出现了：有太多的大师和太多的学徒，我们怎么知道它是否合法？我们都知道，一个大师可以有很多学徒，一个学徒也可能有很多大师，这是合法的。然而，有些“牛”并不诚实，他们持有非法的关系。以HH和3xian为例，HH是3xian的大师，同时3xian是HH的大师，这是非常非法的！为了避免这种情况，请帮助我们判断他们的关系是否合法。请注意，“大师和学徒”关系是可传递的。这意味着如果 A 是 B 的大师，B 是 C 的大师，那么 A 就是 C 的大师。

# Format

## Input
多组数据，每组数据的情况如下：

第一行包含两个整数，N（待测试的成员）和 M（待测试的关系）（2 ≤ N，M ≤ 100）。

接下来是 M 行，每行包含一对（x，y），这意味着 x 是 y 的大师，y 是 x 的学徒。N = 0时表示输入结束。为了简单起见，我们给每个人一个数字（0，1，2，...，N-1），用号码代替他们的名字。

## Output
对于每组数据，如果都是合法的，输出“YES”，否则输出“NO”。

# Samples

```input1
4 3

```

```output1
YES
NO
```
